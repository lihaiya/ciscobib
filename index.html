How Cisco BIB Works

Understand how Cisco BIB recording system uses Built-in Bridge technology for high-quality call recording

BIB Recording Technology

Most Cisco IP phones support BIB functionality. When a call is active, the phone sends its input and output traffic to the voice gateway, then to external PSTN phones (internal/external call recording) or internal users, while copying input and output traffic to the recording server for call recording.

The recording software resides on a server interconnected with the IP phone network and CUCM network, capable of receiving traffic sent by IP phones for recording.

Configuration Guide

Complete Cisco BIB recording system configuration in a few simple steps

Cisco BIB Configuration Steps

1. Phone Interface Configuration

In the Phone configuration interface, select "On" for "Built In Bridge";

2. DN Interface Configuration

In the Directory Number configuration:

  1. In Recording Option, select "Automatic Call Recording Enabled";
  2. In "Recording Profile", select "Recording_Profile";
  3. In "Recording Media Source" dropdown, select "Phone Preferred".
